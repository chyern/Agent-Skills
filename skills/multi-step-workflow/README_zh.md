# 多步骤工作流 (管理者-执行者版)

轻量级 AI Agent 任务追踪工具。该技能为复杂工程任务强制执行专业的 **Manager-Worker (管理者-执行者并行)** 架构。

## 自适应工作流逻辑

Agent 根据任务复杂度进行分流，以最大化效率：

1. **快速路径 (< 3 步)**：针对简单的单项任务。Agent 直接执行。
2. **标准路径 (>= 3 步)**：
   - **第一步：规划模式**：Agent 拟定计划并识别可并行的子任务。**必须停止以等待您的批准**。
   - **第二步：并行执行**：一旦获批，Agent 扮演**管理者**角色。它使用 `spawn` 同时调度最多 **3 个子代理 (Sub-agents)** 处理独立的工人任务，大幅提升吞吐量。

## 为什么需要

单线程 AI 执行大型项目任务太慢。本技能通过 **水平扩展 (Horizontal Scaling)** 解决了这一瓶颈。例如：当需要为 5 个不同模块编写测试时，管理者-执行者模式可以同时处理多个模块。

## 安全与 ClawHub 说明

> [!IMPORTANT]
> **管理者权限**
> 只有主 Agent 负责维护 `task-tracker.js` 的状态。子代理生命周期较短，没有修改全局进度的权限，从而确保状态一致。
>
> **并发限制**
> 默认最大允许 **3 个并行子代理**，以防止 API/CPU 拥塞。

## 脚本说明

| 脚本 | 用途 |

| `task-tracker.js` | 拆分任务为步骤，标记完成，查看进度 |
| `context-snapshot.js` | 在上下文压缩前保存关键发现 |

## 标准用法 (>= 3 步)

1. **分析流程**：Agent 识别任务为“标准路径”。
2. **规划阶段**：Agent 创建步骤并识别并行执行单元。
3. **审批阶段**：Agent 声明“我正处于规划模式”并**停止 (STOP)**。
4. **并行执行**：一旦您说“OK”，管理者 (Manager) 会启动执行者 (Workers) 并协调任务最终完成。

## 手动命令 (可选)

```bash
# 查看所有活动任务和步骤
node scripts/task-tracker.js list

# 标记特定步骤完成
node scripts/task-tracker.js done "任务名" 1
```
